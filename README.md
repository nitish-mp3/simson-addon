# Simson Call Relay вҖ” HAOS Addon

Connects your Home Assistant instance to the Simson VPS control plane for voice/intercom calls between HA instances and Asterisk endpoints.

## Installation

1. Add this repository to Home Assistant as a local addon.
2. Install the **Simson Call Relay** addon.
3. Configure the addon with your VPS server URL, account ID, node ID, and install token.
4. Start the addon.

## Configuration

| Option | Description |
|--------|-------------|
| `server_url` | WSS URL of your Simson VPS (e.g. `wss://simson-vps.niti.life/ws`) |
| `account_id` | Your account ID from the VPS admin |
| `node_id` | This node's unique ID |
| `install_token` | Auth token generated by the VPS admin API |
| `capabilities` | List of capabilities (e.g. `haos`, `voice`, `sip`) |
| `asterisk.enabled` | Enable Asterisk AMI integration |
| `asterisk.host` | Asterisk server host |
| `asterisk.ami_port` | AMI port (default: 5038) |
| `asterisk.ami_user` | AMI username |
| `asterisk.ami_secret` | AMI secret |
| `asterisk.context` | Asterisk dialplan context |
| `asterisk.extension_prefix` | Extension prefix for Simson calls |
| `log_level` | Logging level: debug, info, warning, error |

## Local API Endpoints

The addon exposes a local HTTP API on the ingress port (default 8099):

- `GET /api/health` вҖ” Health check
- `GET /api/status` вҖ” Connection status, node info
- `GET /api/calls` вҖ” Active and recent calls
- `POST /api/call` вҖ” Make a call (`{ "target_node_id": "...", "call_type": "voice" }`)
- `POST /api/answer` вҖ” Answer a call (`{ "call_id": "..." }`)
- `POST /api/reject` вҖ” Reject a call (`{ "call_id": "...", "reason": "busy" }`)
- `POST /api/hangup` вҖ” Hang up a call (`{ "call_id": "..." }`)
